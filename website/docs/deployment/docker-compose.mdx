## Pre-environment

### Docker / Docker Compose

First you need to make sure you have a `Docker / Docker Compose` environment

The installation method can refer to: [Install the docker environment](./install-docker.md)

## Deploy (recommended)

Run on a fresh server:

```bash
curl -fsSL https://raw.githubusercontent.com/AthenDrakomin-hub/tailchat-source/main/scripts/deploy.sh | bash
```

This will:

- Clone/update repo to `/var/www/tailchat-source`
- Ensure `docker-compose.env` exists
- Build images locally via Docker Compose
- Start services and run an optional health check

### Required env

Edit `/var/www/tailchat-source/docker-compose.env` (or pass as env to the deploy script) and make sure these values are set:

- `API_URL`
- `SECRET`
- `ADMIN_PASS`
- `MINIO_ROOT_PASSWORD`
- `DEFENSE_SHARED_SECRET`

## Update

```bash
cd /var/www/tailchat-source
git pull --rebase
docker compose build --pull
docker compose up -d --remove-orphans
docker compose ps
```

The `docker-compose.yml` configuration of `tailchat` provides the following configuration by default:

- `mongodb`: Persistent Database
- `redis`: KV database and message transport service
- `minio`: Distributed file service

The persistent files (database, file storage) are managed uniformly through `docker volume`:

```
docker volume ls | grep "tailchat-server"
```
