安装方式可参考: [安装 Docker 环境](./install-docker.md)

## 部署（推荐）

在新服务器执行：

```bash
curl -fsSL https://raw.githubusercontent.com/AthenDrakomin-hub/tailchat-source/main/scripts/deploy.sh | bash
```

该脚本会：

- 克隆/更新代码到 `/var/www/tailchat-source`
- 确保 `docker-compose.env` 存在
- 通过 Docker Compose 在本机 build 镜像
- 启动服务并执行可选的健康检查

### 必填环境变量

请编辑 `/var/www/tailchat-source/docker-compose.env`（或在执行脚本时通过环境变量传入），至少确保以下字段已配置：

- `API_URL`
- `SECRET`
- `ADMIN_PASS`
- `MINIO_ROOT_PASSWORD`
- `DEFENSE_SHARED_SECRET`

## 更新

```bash
cd /var/www/tailchat-source
git pull --rebase
docker compose build --pull
docker compose up -d --remove-orphans
docker compose ps
```

`tailchat` 的`docker-compose.yml`配置默认提供了如下配置:

- `mongodb`: 持久化数据库
- `redis`: KV数据库与消息中转服务
- `minio`: 分布式文件服务

其中持久化文件(数据库, 文件存储)通过 `docker volume` 统一管理:

```
docker volume ls | grep "tailchat-server"
```
