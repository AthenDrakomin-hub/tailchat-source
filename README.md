# 财富会客厅 (Powered by Tailchat)

## 1. 项目描述

**财富会客厅** 是一款专为高端金融社群、投资理财和首席经济学家对话等场景打造的私密、专属、高度定制化的即时交流平台。

本项目基于新一代开源、高可扩展的即时通讯底座 **Tailchat** 深度定制。旨在为用户提供安全稳定的私密圈子服务，不仅支持常规的即时通讯互动，还通过严格的注册准入机制和独立的安全管理后台，保障整个投资交流社群的合规性与数据私密性。平台整体采用了符合金融调性的暗金与深蓝品牌视觉设计，提供全中文的沉浸式体验。

---

## 2. 核心业务特性

Web端（用户界面）为广大投资者和社群成员提供了流畅、高效的交互体验：

* **专属身份认证与准入**
  * **多方式登录/注册**：支持手机号与邮箱的无缝切换登录和注册。
  * **私密准入机制**：注册账号必须提供有效的“组织代码”（如：`0813` 或 `朝闻道`），有效拦截无关人员，保障社群纯净度。
  * **专属社群环境**：所有账号密码遗失等问题统一由超级管理员在后台协助找回，保障用户信息安全与统一管理。

* **社群即时通讯**
  * **多频道沟通**：支持在一个社群内划分不同的文本频道（如：大盘分析、个股交流、宏观数据发布等）。
  * **富文本与多媒体**：支持发送文字、表情、图片、文件以及富文本消息。
  * **消息通知与免打扰**：支持针对特定频道的精准消息推送与免打扰设置。

* **私信语音通话（语音模式）**
  * **私信一键发起通话**：在私信会话中可一键发起语音通话。
  * **禁用群语音入口**：群组内不提供语音频道/群语音入口，避免误用为群聊语音房。
  * **默认语音**：默认以语音通话为主；如业务确有需要也可开启视频能力（可通过配置开关强制纯语音）。

* **伪直播（MP4 → HLS）**
  * **群管理发起直播**：群管理在文本频道内上传 MP4 并开播，系统自动转码为 HLS 分片流。
  * **直播卡片消息**：开播后自动在频道内发送“直播卡片”，成员点击即可弹窗观看。
  * **弱化文件暴露**：对外提供的是 HLS 分片流，不直接暴露 MP4 文件直链（仍可被录屏/抓包，属业务可接受范围）。

* **高效社群管理（管理员专属）**
  * **一键全群禁言**：频道管理员专属的“一键麦克风”快捷开关，可瞬间开启或解除当前频道的全员禁言，轻松应对突发情况或重要发布时刻。
  * **成员管理**：在频道内可查看成员列表、授予权限或移除违规用户。

* **极致品牌体验**
  * **专属视觉UI**：全站采用深蓝底色搭配暗金交互（如按钮、高亮状态），配备首席经济学家专属的高清登录海报。
  * **全中文纯净体验**：系统强制全局中文显示，移除了多余的语言切换、系统状态监控等不必要的繁杂选项，界面更加清爽专注。

## 3. 底层架构与核心概念 (Architecture & Core Concepts)

本项目不仅仅是一个简单的聊天室，底层继承了 Tailchat 强大的基础架构能力：

* **“前端微内核 + 后端微服务”双微架构**
  * **前端**：基于 `mini-star` 的微内核架构，允许在不刷新页面的情况下动态加载/卸载插件 UI。
  * **后端**：基于 `Moleculer` 框架构建，各个服务（用户、群组、聊天、网关）通过内部 Transporter（Redis）通信，具备极高的横向扩展能力。
* **多工作区模型（Workspace）**
  * 采用类似 Discord 的群组（Guild）和面板（Panel）树状体系，不仅支持文本，底层还支持 Webview 嵌入、自定义 HTML 等多种插件类型的频道扩展。
* **完善的 RBAC 权限体系**
  * 底层支持极细粒度的身份组（Role）配置和灵活的权限位控制，为“超级管理员”与“频道管理员”的权限隔离提供坚实基础。
* **插件化生态（Plugin System）**
  * 系统中的许多特殊功能（如伪直播、一键禁言）均可通过插件形式实现与挂载，保证核心代码的纯净与系统的高可扩展性。
* **全端覆盖支持**
  * 底层代码库包含了对 Web、Windows/macOS (Electron) 以及 iOS/Android (React Native) 的全端支持能力。

## 4. 管理端功能

管理端作为独立部署的安全服务模块，为平台运营者提供了上帝视角的超级管理权限，与前台用户业务物理隔离：

* **独立安全入口**
  * 拥有独立的管理端登录地址（`/admin/`）和专属的超管账号密码，与普通用户数据库彻底隔离，防止越权操作。

* **用户与账号管理**
  * **全局用户查询**：可查看平台内所有已注册的用户信息。
  * **账号封禁与解封**：对违规操作的用户进行全局账号封禁或解封。
  * **信息重置**：帮助丢失凭证的用户修改基础信息或密码。

* **群组与频道监控**
  * **全局社群管理**：查看平台上所有建立的社群（群组）及下属频道结构。
  * **内容监管**：可介入查看或清理不合规的频道和群组信息。

* **系统全局配置**
  * **基础信息设置**：修改平台的全局名称、基础介绍等核心参数。
  * **注册与邀请策略**：在后台全局开启或关闭新用户注册通道。
  * **系统公告下发**：可向全平台所有在线用户下发置顶的系统级通知和公告。

## 5. Linux 服务器完整部署与编译指南 (推荐)

对于生产环境，官方强烈推荐基于 Linux 服务器使用 Docker Compose 进行完整拉取、编译与部署。以下是从 0 到 1 的标准流程。

### 5.1 服务器环境准备
建议服务器配置：**2核4G** 及以上（编译过程对内存有一定要求）。
需提前安装以下基础环境：
- **Node.js** (v16.18.0+) & **pnpm** (v8.3.1+)
- **Git**
- **Docker** & **Docker Compose**

### 5.2 克隆项目仓库
登录服务器并拉取项目源码：
```bash
mkdir -p /opt/msgbyte && cd /opt/msgbyte
git clone https://github.com/AthenDrakomin-hub/tailchat-source.git
cd tailchat-source

# (可选) 切换到稳定的 release tag
# git checkout v1.x.x
```

### 5.3 配置文件准备
项目根目录下的 `docker-compose.yml` 和 `docker-compose.env` 是 Docker 部署的核心。
在编译启动前，请务必编辑 `docker-compose.env` 文件：

```bash
nano docker-compose.env
```

**必须修改/确认的核心配置项：**
- `SECRET`：JWT 签名密钥（必须设置，建议使用随机长字符串）。
- `API_URL`：外部访问地址（必须设置，需与实际访问域名/端口一致，例如 `https://chat.example.com` 或 `http://<服务器IP>:11000`）。**注意：如果不设置此项，会导致发送的图片等文件无法正常显示。**
- `ADMIN_USER` / `ADMIN_PASS`：管理端账号密码（必须设置）。
- `MINIO_ROOT_USER` / `MINIO_ROOT_PASSWORD`：对象存储管理账号密码（建议修改默认值）。
- `FILE_LIMIT`：全局文件上传大小限制（单位：字节，默认为 104857600 即 100MB）。

### 5.4 编译与构建 (Build)
在项目根目录执行 Docker 构建命令。该过程会自动拉取基础镜像、安装所有前后端依赖，并并行打包核心服务、管理端以及所有微内核插件。

```bash
# 构建所有服务镜像 (耗时约 5-10 分钟，取决于服务器性能)
docker compose build
```

*注意：如果修改了前端静态代码（如 `consts.ts` 中的文件上传限制），必须重新执行此 build 命令以生成最新的前端产物。*

### 5.5 启动服务 (Up)
构建完成后，一键启动所有微服务及底层依赖（MongoDB, Redis, MinIO）：

```bash
# 后台守护模式启动
docker compose up -d
```

### 5.6 访问地址与网关路由
系统内置 Traefik 作为动态网关，对外暴露的服务路由如下（请确保云服务器防火墙已放行 `11000` 端口）：
- **Web 端**：`http://<服务器IP>:11000/`
- **管理端**：`http://<服务器IP>:11000/admin/`
- **OpenAPI 接口**：`http://<服务器IP>:11000/open/` (内部映射至 3003 端口)

*注意：若在生产环境中接入 Nginx 或云厂商 Load Balancer，请务必确保开启 WebSocket 支持并合理设置长连接超时时间，以保证 IM 通信的稳定性。*

## 6. 高阶运维与客户端构建 (Advanced Operations & Client Build)

### 6.1 服务拆分与横向扩容
通过修改 `docker-compose.yml` 中各容器的 `SERVICES` 环境变量（如 `core/gateway,core/user/*.service.js`），运维人员可以在多台物理机上拆分部署不同的微服务模块，配合 Redis Transporter 实现高可用与负载均衡。

### 6.2 客户端构建指引
项目代码库中包含了全端客户端的源码：
- **桌面端 (Desktop)**：进入 `client/desktop` 目录，基于 Electron 构建 Windows/macOS 安装包。
- **移动端 (Mobile)**：进入 `client/mobile` 目录，基于 React Native 构建 iOS/Android 应用程序。

### 6.3 运维脚本
* **普通账号 JWT 自动发话术**
  * 脚本：`scripts/pseudolive-bot.ts`
  * 配置示例：`scripts/pseudolive-bot.example.json`
* **数据备份**：强烈建议定期对 Docker Volume 中的 MongoDB 数据卷和 MinIO 存储卷进行快照备份，以防数据丢失。
